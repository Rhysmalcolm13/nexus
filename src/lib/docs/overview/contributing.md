# Contributing Guide

## Code Style

- TypeScript strict mode
- ESLint configuration
- Prettier formatting
- Component structure
- File naming conventions

## Testing

- Unit tests
- Integration tests
- E2E tests
- Test coverage requirements

## Documentation

- JSDoc comments
- README files
- API documentation
- Component documentation

## Review Process

1. Create feature branch
2. Write tests
3. Update documentation
4. Create pull request
5. Code review
6. Merge to main
